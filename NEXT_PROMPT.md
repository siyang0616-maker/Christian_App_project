# 지금 세션에서 코덱스에게 줄 프롬프트

아래 내용을 그대로 코덱스에게 전달한다.
SESSION_START_CHECKLIST.md와 LEADER_CARE_ROADMAP.md 두 파일을
프로젝트 루트(supabase 폴더, app 폴더와 같은 위치)에 먼저 복사해 넣은 뒤
이 프롬프트를 사용한다.

---

이 프로젝트는 윈도우와 맥을 번갈아 쓴다. 앞으로 모든 세션에서
SESSION_START_CHECKLIST.md를 가장 먼저 끝까지 실행한다.
지금 막 그 파일을 프로젝트 루트에 추가했다. 지금부터 그 체크리스트의
1~5단계를 순서대로 실행하고 결과를 보여줘라.

체크리스트 결과 특별한 문제가 없다면 (push 안 된 커밋 없음, 충돌 없음,
.env.local 존재), 이어서 다음을 진행한다.

## 지금 상태
- A (스모크 테스트 1-6단계): 완료
- B (GitHub 인증): gh CLI 미설치로 막힘
- C/D/E (티켓 A/B/C): LEADER_CARE_ROADMAP.md가 없어서 막혔었음 →
  지금 그 파일을 프로젝트 루트에 추가했다.

## B 먼저 처리
체크리스트 3단계에서 확인한 현재 OS에 맞는 GitHub CLI 설치 명령어를
정확히 안내하고 멈춰라. 내가 직접 설치와 로그인을 진행한 뒤
"설치 완료"라고 말하면 git push origin main을 재시도해라.
설치를 못 했어도 아래 C/D/E 작업은 계속 진행한다 (push만 보류).

## C. 티켓 A — 컨택 로그
LEADER_CARE_ROADMAP.md의 "티켓 A" 섹션을 그대로 구현해라.
완료 기준:
- corepack pnpm lint / typecheck / build 전부 raw 출력 그대로 보여주기
- docs/WORKLOG.md에 "- [현재OS] Ticket A 컨택 로그 구현" 한 줄 추가
- git add -A && git commit -m "feat: contact-waiting badge (ticket A)"
- git push 시도 (B가 해결됐으면 실제로, 안 됐으면 결과만 기록하고 계속 진행)

## D. 티켓 B — 키워드 신호 감지 (C가 끝난 뒤)
LEADER_CARE_ROADMAP.md의 "티켓 B" 섹션을 그대로 구현해라.
반드시 지킬 것:
- 위험도 점수나 % 같은 수치 평가를 절대 만들지 마라. boolean만 사용한다.
- 구현 후 테스트 멤버 계정으로 실제로 로그인해서, 이 표시가 멤버 화면
  어디에도 보이지 않는 것을 직접 눈으로 확인하고 그 결과를 보고해라.
  "안 보일 것으로 예상된다"가 아니라 실제로 들어가서 확인한 내용만 적어라.
완료 기준은 C와 동일한 형식 (lint/typecheck/build, WORKLOG, commit).

## E. 티켓 C — 성장 타임라인 (D가 끝난 뒤)
LEADER_CARE_ROADMAP.md의 "티켓 C" 섹션을 그대로 구현해라.
28칸 그리드가 실제 체크인 데이터와 정확히 일치하는지 테스트 계정으로
직접 들어가서 확인하고 보고해라.
완료 기준은 C와 동일.
