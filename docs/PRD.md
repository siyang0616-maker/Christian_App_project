# Donghaeng Room 1-Week MVP PRD

## 1. 제품 요약

동행방은 소그룹을 위한 모바일-first 신앙 체크인 및 기도제목 기록 앱이다.

핵심 메시지:

> 카톡에 흘러가는 기도제목을, 함께 기억하는 기록으로.

동행방은 카카오톡을 대체하지 않는다. 실시간 대화방, 성경 콘텐츠 앱, AI 묵상 앱, 교회 전체 관리 시스템도 아니다. 1주 MVP의 목표는 소그룹 리더가 다음 주에도 다시 열어보고 싶은지 검증하는 것이다.

## 2. 4-Role Product Loop

### Developer

- 중요: Next.js App Router, TypeScript, Tailwind, Supabase Auth/Postgres/RLS, Vercel Hobby 안에서 단순하게 구현한다.
- 위험: 기도제목과 체크인은 민감 정보이므로 UI 조건만으로 권한을 막으면 안 된다.
- 지금 포함: 인증, 프로필, 그룹 생성, 초대코드 참여, 체크인, 기도제목 카드, 기도했어요 반응, 리더 대시보드.
- 보류: 실시간 채팅, 파일 업로드, 푸시, AI, 결제, 교회 전체 관리자 기능.

### Entrepreneur

- 중요: 첫 비즈니스 목표는 매출이 아니라 리더 재사용 의향 검증이다.
- 위험: 기능이 많아질수록 1주 안에 베타 검증하기 어렵다.
- 지금 포함: 리더가 멤버 흐름을 돌봄 관점으로 확인하고 기도제목을 잊지 않게 하는 최소 기능.
- 보류: 주간 요약, 커스텀 체크인 항목, 유료 플랜, 부서 단위 리포트.

### Web Designer

- 중요: 첫 사용자가 설명 없이 20초 안에 체크인할 수 있어야 한다.
- 위험: 리더 화면이 감시, 평가, 출석 관리처럼 보이면 멤버가 안전하게 느끼지 못한다.
- 지금 포함: 큰 터치 영역, 부드러운 카드 UI, 공개 범위 안내, 한국어 돌봄 카피.
- 보류: 복잡한 통계, 점수, 랭킹, 스트릭, 무거운 종교 포스터 스타일.

### Marketer

- 중요: “카톡 대체”가 아니라 “카톡에 흘러가는 체크인과 기도제목을 기억하는 앱”으로 말한다.
- 위험: 교회 관리 시스템처럼 들리면 소그룹 리더의 가벼운 베타 도입 장벽이 높아진다.
- 지금 포함: 리더 가치 제안, 멤버의 안전한 공유, 베타 테스트 메시지.
- 보류: 과장된 영적 성과 주장, 목회 돌봄 대체 표현, 결제 유도.

## 3. MVP 목표

1주 MVP의 핵심 검증 질문:

> 소그룹 리더가 다음 주에도 동행방을 다시 열어보고 싶은가?

성공 기준:

- 리더가 초대코드로 실제 소그룹 멤버를 초대한다.
- 멤버가 하루 20초 안에 체크인을 남긴다.
- 기도제목이 카드로 남아 리더와 멤버가 다시 볼 수 있다.
- 리더가 조용한 멤버와 기도로 기억할 제목을 확인한다.
- 리더가 “카톡보다 기도제목을 기억하기 쉬웠다”고 말한다.

## 5. 핵심 기능

### Auth

- 이메일/비밀번호 기반 Supabase Auth.
- MVP에서는 소셜 로그인, SMS 인증, 교회 SSO 제외.

### Profile Setup

- 표시 이름만 입력한다.
- 프로필 이미지 업로드 제외.

### Create Group

- 리더가 동행방 이름을 입력하고 그룹을 만든다.
- 생성자는 리더 역할을 가진다.
- 초대코드가 생성된다.

### Join by Invite Code

- 멤버는 리더에게 받은 초대코드로 참여한다.
- 초대코드 조회는 `join_group_by_code` RPC로 처리해 그룹 목록이 노출되지 않게 한다.

### Daily Check-In

- 멤버는 오늘의 마음 상태를 선택하고 한 줄 나눔을 남긴다.
- 리듬 예시: 일어났어요, 말씀 읽었어요, 기도했어요, 묵상했어요, 예배/모임에 참석했어요.
- 상태 예시: 좋음, 보통, 힘듦, 기도 필요.
- 공개 범위: 나만 보기, 리더와 나, 소그룹 전체.

### Prayer Request Cards

- 기도제목은 채팅 메시지가 아니라 카드로 남긴다.
- 공개 범위: 나만 보기, 리더와 나, 소그룹 전체, 익명.

### “기도했어요” Reaction

- 긴 댓글이나 채팅 없이 기도로 기억했다는 신호만 남긴다.
- 같은 사용자는 같은 기도제목에 한 번만 반응한다.

### Leader Dashboard

- 리더는 멤버 수, 기도로 기억할 제목, 조용한 멤버, 최근 체크인을 본다.
- 문구는 돌봄 중심으로 유지한다.
- 금지 표현: 미참여자, 불성실, 실패, 벌점, 관리 대상, 감시, 체크 안 한 사람.

## 6. 한국어 UX 카피 예시
