# Donghaeng Room Project Status

작성일: 2026-06-16

## 현재까지 프롬프트로 정리된 지점

지금까지의 프롬프트와 작업 결과는 "소그룹 리더가 다음 주에도 다시 열어보고 싶은 말씀·기도 체크인 MVP"까지 구체화되어 있습니다.

정리된 제품 방향:

- 제품명: 동행방 / Donghaeng Room
- 대상: 소그룹 리더, 셀장, 순장, 목장 리더, 청년부 리더, 제자훈련 리더, 소그룹 멤버
- 포지셔닝: 카카오톡 대체 채팅 앱이 아니라, 카톡에 흘러가는 체크인과 기도제목을 기록으로 남기는 앱
- 핵심 검증 질문: 리더가 다음 주에도 동행방을 다시 열어보고 싶은가?
- MVP 운영 목표: 2주 안에 3~5명의 실제 소그룹 리더와 베타 테스트

정리된 기능 범위:

- 이메일 기반 로그인/회원가입
- 프로필 이름 설정
- 동행방 생성
- 초대코드로 동행방 참여
- 오늘의 신앙 리듬 체크인
- 기도제목 카드
- 공개 범위 선택
- "기도했어요" 반응
- 리더 돌봄 보드
- Supabase RLS 기반 개인정보 보호

정리된 제외 범위:

- 풀 채팅
- AI 묵상/추천
- 결제
- 성경 콘텐츠 DB
- 푸시 알림
- 파일 업로드
- 네이티브 앱
- 교회 전체 관리자 기능
- 순위, 점수, 스트릭

## 현재 코드 구현 상태

이미 구현된 흐름:

- `/`: 인증 상태에 따라 로그인, 프로필 설정, 그룹 생성/참여, 메인 체크인 루프로 분기
- `/today`: 오늘 체크인 중심 화면
- `/leader`: 리더 전용 돌봄 보드
- 서버 액션: auth, profile, group, check-in, prayer
- Supabase 마이그레이션: profiles, groups, group_members, checkins, prayers, prayer_reactions, RLS policies
- UI 컴포넌트: 앱 쉘, 인증 패널, 체크인 폼, 기도제목 폼/목록, 리더 대시보드

GitHub 준비 상태:

- `.gitignore`에 `.env.local`, `.next`, `node_modules`, 로그 파일, 빌드 산출물이 제외되어 있음
- `.env.example`이 있음
- README가 GitHub 첫 화면 기준으로 정리됨
- 제품 문서가 `docs/`에 분리되어 있음
