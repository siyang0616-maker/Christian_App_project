# Donghaeng Room User Flow

## 1. UX 목표

멤버는 길게 설명하지 않아도 20초 안에 오늘의 신앙 리듬을 남길 수 있어야 한다. 리더는 멤버를 평가하는 화면이 아니라 함께 기억하고 돌보기 위한 흐름을 보아야 한다.

## 2. 4-Role Flow Review

### Developer

- 중요: 각 단계가 서버 액션과 RLS 정책으로 보호되어야 한다.
- 위험: UI에서만 role이나 visibility를 판단하면 민감 정보가 노출될 수 있다.
- 지금 구축: 인증 상태 기반 분기, 프로필 확인, 그룹 멤버십 확인, 공개 범위 저장.
- 보류: 복잡한 라우팅, 실시간 채팅방, 알림 기반 복귀 흐름.

### Entrepreneur

- 중요: 리더가 3~5명 베타에서 설명 없이 초대하고 사용할 수 있어야 한다.
- 위험: 온보딩이 길면 리더가 베타 그룹에 적용하기 어렵다.
- 지금 구축: 한 화면 중심의 핵심 루프.
- 보류: 세부 튜토리얼, 조직 설정, 결제 전환.

### Web Designer

- 중요: 모바일에서 각 화면의 다음 행동이 분명해야 한다.
- 위험: 공개 범위가 숨겨지면 멤버가 기도제목을 안전하게 나누기 어렵다.
- 지금 구축: 카드, 큰 버튼, 짧은 텍스트, 공개 범위 helper copy.
- 보류: 복잡한 메뉴, 긴 설명, 통계 중심 UI.

### Marketer

- 중요: 사용 흐름 자체가 “카톡에 흘러가지 않음”을 느끼게 해야 한다.
- 위험: 앱이 대화 앱처럼 보이면 카톡과 직접 경쟁하게 된다.
- 지금 구축: 체크인과 기도제목을 기록으로 남기는 흐름.
- 보류: 채팅, 피드, 콘텐츠 추천.

## 3. First-Time Leader Flow

1. 로그인 또는 계정 생성
2. 프로필 이름 입력
3. 새 동행방 만들기
4. 동행방 이름 입력
5. 초대코드 확인
6. 멤버에게 초대 메시지 공유
7. 오늘 체크인 남기기
8. 기도제목 카드 남기기
9. 리더 돌봄 보드 확인

화면 카피 예시:

- 헤더: 오늘의 신앙 리듬
- 그룹 생성: 새 동행방 만들기
- 설명: 리더로 소그룹을 시작해요.
- 초대: 초대코드 `AB12CD`
- 대시보드: 멤버의 흐름을 돌봄 관점으로 살펴봐요.

## 4. First-Time Member Flow

1. 로그인 또는 계정 생성
2. 프로필 이름 입력
3. 초대코드 입력
4. 오늘 체크인 선택
5. 한 줄 나눔 입력
6. 공개 범위 선택
7. 기도제목 카드 작성
8. “기도했어요” 반응으로 함께 기억하기

화면 카피 예시:

- 체크인 제목: 오늘 체크인
- 설명: 20초 안에 남기는 오늘의 신앙 리듬
- 한 줄 나눔 placeholder: 오늘 마음에 남은 말씀, 감사, 안부를 짧게 남겨요.
- 공개 범위 helper: 제출 전에 공개 범위를 꼭 확인해 주세요

## 5. Daily Check-In Flow

입력 순서:

1. 오늘 리듬 체크
2. 마음 상태 선택
3. 한 줄 나눔 작성
4. 공개 범위 선택
5. 저장

상태 옵션:

- 좋음
- 보통
- 힘듦
- 기도 필요

완료 카피:

- 오늘은 좋음
- 짧게 남겨도 충분해요. 함께 기억할 수 있는 하루의 마음이면 돼요.

## 6. Prayer Request Flow

입력 순서:

1. 기도제목 작성
2. 공개 범위 선택
3. 기도제목 남기기
4. 멤버가 “기도했어요” 반응

카피 예시:

- 제목: 함께 기억할 기도제목
- 설명: 카톡처럼 흘러가지 않게 카드로 남겨요.
- placeholder: 기도로 함께 기억하고 싶은 제목을 남겨요.
- 반응: 기도했어요
- 반응 완료: 기도로 기억 중

## 7. Leader Dashboard Flow

리더가 확인하는 정보:

- 함께하는 멤버 수
- 기도로 기억할 제목 수
- 조용한 멤버
- 최근 체크인

카피 원칙:

- 사용: 조용한 멤버, 안부가 필요할 수 있어요, 함께 기억해요.
- 금지: 미참여자, 불성실, 실패, 벌점, 관리 대상, 감시, 체크 안 한 사람.

## 8. Manual UX Test

- 신규 리더가 설명 없이 그룹을 만들 수 있는가?
- 멤버가 20초 안에 체크인을 완료할 수 있는가?
- 기도제목 작성 전 공개 범위가 눈에 보이는가?
- 리더 대시보드가 관리 화면이 아니라 돌봄 화면처럼 느껴지는가?
- 카톡보다 기도제목을 다시 찾기 쉬운가?
