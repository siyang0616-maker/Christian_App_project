# Donghaeng Room Beta Test Guide

## 1. Beta Goal

2주 안에 3~5명의 소그룹 리더와 실제 그룹으로 검증한다.

핵심 질문:

> 리더가 다음 주에도 동행방을 다시 열어보고 싶은가?

기능 수보다 리더 재사용 의향이 중요하다.

## 2. 4-Role Beta Review

### Developer

- 중요: 베타는 Vercel Hobby와 Supabase Free Plan으로 운영한다.
- 위험: 테스트 중 권한 오류나 RLS 누락이 있으면 신뢰를 잃는다.
- 지금 구축: 기본 배포, DB 정책, 모바일 웹 사용.
- 보류: 네이티브 앱, 푸시, 유료 분석, 복잡한 운영 도구.

### Entrepreneur

- 중요: 리더가 실제 소그룹에 적용할 만큼 문제가 큰지 확인한다.
- 위험: “좋네요”라는 반응만 듣고 재사용 의향을 확인하지 않는 것.
- 지금 구축: 2주 베타 인터뷰와 핵심 지표.
- 보류: 가격 확정, 교회 전체 계약, 기능 확장 로드맵.

### Web Designer

- 중요: 멤버가 부담 없이 체크인하고 기도제목을 남길 수 있어야 한다.
- 위험: UI가 관리/평가처럼 느껴지면 멤버 입력이 줄어든다.
- 지금 구축: 모바일 기준 사용성 질문.
- 보류: 데스크톱 최적화, 고급 대시보드.

### Marketer

- 중요: 베타 메시지는 “카톡 대체”가 아니라 “흘러가는 기도제목을 기억”으로 말한다.
- 위험: 너무 큰 비전으로 말하면 초기 리더가 부담을 느낀다.
- 지금 구축: 리더 모집 문구, 초대 문구, 피드백 질문.
- 보류: 유료 전환 캠페인, 큰 교회 대상 세일즈 자료.

## 3. Beta Participant Profile

우선 모집:

- 매주 소그룹을 운영하는 리더
- 카톡에서 기도제목을 찾기 어렵다고 느낀 리더
- 멤버 안부를 챙기고 싶지만 일일이 묻기 부담스러운 리더
- 청년부, 셀, 순, 목장, 제자훈련 소그룹 리더

베타 제외:

- 교회 전체 출석 관리 도구를 원하는 사용자
- 실시간 채팅 대체를 원하는 사용자
- 설교/성경 콘텐츠 앱을 원하는 사용자

## 6. Beta Test Script

리더에게 요청할 행동:

1. 동행방 만들기
2. 초대코드 공유
3. 본인 체크인 1회 남기기
4. 기도제목 카드 1개 남기기
5. 멤버에게 3일 이상 체크인 요청
6. 리더 돌봄 보드 확인
7. 1주 뒤 다시 열어볼 의향 답변

멤버에게 요청할 행동:

1. 초대코드로 참여
2. 프로필 이름 입력
3. 오늘 체크인 남기기
4. 공개 범위 선택
5. 기도제목 카드 남기기
6. 다른 기도제목에 “기도했어요” 반응

## 7. Success Metrics

핵심 지표:

- 5명 중 3명 이상의 리더가 2주차에 다시 사용 의향을 보인다.

보조 지표:

- 각 그룹에 4명 이상 참여.
- 각 그룹에서 체크인 10개 이상 생성.
- 각 그룹에서 기도제목 카드 3개 이상 생성.
- “기도했어요” 반응이 그룹당 5개 이상 생성.
- 리더가 대시보드를 2회 이상 확인.

정성 지표:

- “카톡보다 기도제목을 다시 찾기 쉬웠다.”
- “멤버 안부를 묻는 부담이 조금 줄었다.”
- “부담스럽기보다 함께 기억되는 느낌이었다.”

## 8. Interview Questions

리더 질문:

- 이번 주에 동행방을 다시 열어본 순간이 있었나요?
- 리더 돌봄 보드가 실제 안부 확인에 도움이 되었나요?
- 카톡만 쓸 때보다 기도제목을 기억하기 쉬웠나요?
- 어떤 순간에 앱이 부담스럽게 느껴졌나요?
- 다음 주에도 이 소그룹에서 계속 써볼 의향이 있나요?
- 이 기능이 유료라면 어떤 조건에서 검토할 수 있나요?

멤버 질문:

- 체크인이 20초 안에 끝났나요?
- 공개 범위가 충분히 분명했나요?
- 기도제목을 남길 때 안전하다고 느꼈나요?
- “기도했어요” 반응이 부담 없이 느껴졌나요?
- 카톡에 쓰는 것보다 더 편한 점이나 불편한 점은 무엇이었나요?
