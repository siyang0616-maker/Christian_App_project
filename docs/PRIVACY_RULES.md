# Donghaeng Room Privacy Rules

## 1. Privacy Principle

기도제목과 체크인은 민감한 신앙·정서 데이터다. 동행방은 멤버가 감시받는 느낌 없이 자신의 상태를 나눌 수 있어야 한다.

핵심 원칙:

- UI-only permission check 금지.
- Supabase RLS로 접근 제한 강제.
- service_role key 노출 금지.
- 공개 범위는 입력 화면에서 명확히 보여준다.
- 익명 기도제목은 UI에서 작성자 이름이 드러나지 않게 한다.

## 2. 4-Role Privacy Review

### Developer

- 중요: 모든 민감 데이터 조회는 RLS 정책으로 제한한다.
- 위험: 앱 코드 실수 하나로 private/leader 콘텐츠가 노출될 수 있다.
- 지금 구축: checkins, prayers에 visibility 컬럼과 RLS helper 적용.
- 보류: 세밀한 감사 로그, 관리자 보안 콘솔.

### Entrepreneur

- 중요: 베타 리더와 멤버의 신뢰가 제품 검증의 전제다.
- 위험: 한 번의 개인정보 노출은 재사용 의향을 크게 떨어뜨린다.
- 지금 구축: 낮은 기능 수, 높은 개인정보 보호 명확성.
- 보류: 공유 링크 공개 페이지, 외부 분석 도구.

### Web Designer

- 중요: 멤버가 작성 전에 “누가 볼 수 있는지” 바로 알아야 한다.
- 위험: 공개 범위가 숨겨지면 기도제목 작성이 부담스러워진다.
- 지금 구축: 공개 범위 select와 “제출 전에 공개 범위를 꼭 확인해 주세요” 카피.
- 보류: 복잡한 권한 설정 화면.

### Marketer

- 중요: 안전한 나눔은 동행방의 핵심 가치다.
- 위험: “리더가 다 볼 수 있다”는 인상을 주면 감시 앱처럼 느껴진다.
- 지금 구축: 돌봄, 기억, 리듬 중심 메시지.
- 보류: 참여율, 순위, 성과 추적 메시지.

## 3. Visibility Copy

공개 범위 옵션:

- 나만 보기: 개인 기록으로 남겨요.
- 리더와 나: 리더만 함께 기억해요.
- 소그룹 전체: 모든 멤버가 볼 수 있어요.
- 익명으로: 이름 없이 함께 기도해요. 기도제목에서만 사용해요.

Helper copy:

- 제출 전에 공개 범위를 꼭 확인해 주세요.

## 4. RLS Enforcement

### Profiles

- 본인 프로필은 본인이 생성/수정한다.
- 같은 그룹 멤버끼리만 표시 이름을 볼 수 있다.

### Groups

- 그룹 멤버 또는 생성자만 그룹 정보를 볼 수 있다.
- 초대코드 검색을 위해 그룹 select를 열지 않는다.

### Group Members

- 그룹 멤버는 같은 그룹의 멤버십을 볼 수 있다.
- 리더 멤버십 생성은 그룹 생성자에게만 허용한다.
- 일반 멤버 참여는 `join_group_by_code` RPC로 처리한다.

### Check-Ins

- 작성자는 본인 체크인을 볼 수 있다.
- `group`: 그룹 멤버가 볼 수 있다.
- `leader`: 작성자와 리더가 볼 수 있다.
- `private`: 작성자만 볼 수 있다.

### Prayer Requests

- 작성자는 본인 기도제목을 볼 수 있다.
- `group`, `anonymous`: 그룹 멤버가 볼 수 있다.
- `leader`: 작성자와 리더가 볼 수 있다.
- `private`: 작성자만 볼 수 있다.

### Prayer Reactions

- 사용자는 볼 수 있는 기도제목에만 반응을 볼 수 있다.
- 사용자는 볼 수 있는 기도제목에만 “기도했어요”를 추가할 수 있다.

## 5. UX Safety Rules

사용할 표현:

- 조용한 멤버
- 안부가 필요할 수 있어요
- 함께 기억해요
- 기도로 기억할 제목
- 오늘의 신앙 리듬

피할 표현:

- 미참여자
- 불성실
- 실패
- 벌점
- 관리 대상
- 감시
- 체크 안 한 사람

## 6. Operational Rules

- `.env`, `.env.local`, service_role keys는 커밋하지 않는다.
- Vercel에는 anon key만 설정한다.
- 외부 분석 스크립트는 MVP에서 사용하지 않는다.
- 파일 업로드와 프로필 이미지는 MVP에서 제외한다.
- 카카오 알림 API, SMS, push notification은 MVP에서 제외한다.

## 7. Manual Privacy Test

- private 체크인이 다른 멤버에게 보이지 않는가?
- leader 체크인이 리더와 작성자에게만 보이는가?
- anonymous 기도제목에서 작성자 이름이 UI에 나오지 않는가?
- 초대코드 없이 그룹 목록을 조회할 수 없는가?
- 멤버가 다른 그룹의 기도제목을 볼 수 없는가?
