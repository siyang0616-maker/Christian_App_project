# Beta Test Plan Skill

## Purpose

Prepare a small, safe beta test that answers whether leaders want to use Donghaeng Room again next week.

## When To Use

- Before inviting external leaders.
- After QA smoke tests pass.
- When updating beta instructions or feedback forms.

## Inputs Required

- Beta URL.
- Target leader count.
- Target member count.
- Test duration.
- Known limitations.
- Feedback collection method.

## Step-By-Step Process

1. Define the beta goal.
2. Select the smallest useful test group.
3. Prepare leader invite copy.
4. Prepare member invite copy.
5. Define day-by-day usage asks.
6. Define quantitative and qualitative signals.
7. Define stop, improve, continue, and expand criteria.
8. Confirm privacy instructions for sensitive prayer content.

## Output Format

```md
## Beta Test Plan

Goal:
- ...

Participants:
- ...

Schedule:
- ...

Instructions:
- Leader:
- Member:

Success Signals:
- ...

Risks:
- ...
```

## Acceptance Criteria

- Starts with 1 trusted leader or a very small group unless larger beta is justified.
- Uses public beta URL, not localhost.
- Includes privacy guidance.
- Defines clear continue/improve/pivot signals.
