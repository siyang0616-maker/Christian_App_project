# Competitor Differentiation Skill

## Purpose

Prevent product drift by comparing each idea against the comparison product set.

## When To Use

- Before approving any new feature.
- During roadmap review.
- When a request sounds like chat, community, Bible content, devotional content, or church admin.

## Inputs Required

- Proposed feature or change.
- Target user.
- User problem.
- Expected benefit.

## Step-By-Step Process

1. Describe the proposed feature.
2. Compare it against each comparison product.
3. Identify whether the feature competes where Donghaeng Room cannot win.
4. Identify whether it strengthens the wedge: check-in, prayer memory, leader care.
5. Decide: build, narrow, manual test, defer, or reject.
6. Record the reasoning in a decision record if the decision affects roadmap scope.

## Output Format

```md
## Competitor Differentiation Review

Proposal:
- ...

Comparison:
- KakaoTalk:
- Naver BAND:
- YouVersion:
- Hallow / Glorify / Pray.com:
- Planning Center / Church Center:

Decision:
- ...

Reason:
- ...
```

## Acceptance Criteria

- Mentions all five comparison groups.
- Clearly states whether the idea strengthens the unique wedge.
- Recommends narrowing or rejecting if the idea imitates a larger product.
- Preserves MVP boundaries.
