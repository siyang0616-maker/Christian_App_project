# Retention Review Skill

## Purpose

Evaluate whether the current MVP creates weekly reuse for leaders and repeated safe participation for members.

## When To Use

- Before beta to define retention hypotheses.
- After beta to interpret feedback.
- Before roadmap updates that claim to improve engagement.

## Inputs Required

- Current user flow.
- Beta observations or expected behavior.
- Leader feedback.
- Member feedback.
- Usage counts if available.

## Step-By-Step Process

1. State the leader reuse hypothesis.
2. State why a member would check in more than once.
3. Identify the screen or moment that creates repeated value.
4. Identify the first-week aha moment.
5. Identify likely drop-off points.
6. Separate retention blockers from feature wish lists.
7. Recommend the smallest next action.

## Output Format

```md
## Retention Review

Leader Return Reason:
- ...

Member Repeat Reason:
- ...

Repeated Value Screen:
- ...

First-Week Aha:
- ...

Likely Drop-Off:
- ...

Next Action:
- ...
```

## Acceptance Criteria

- Ties retention to check-in, prayer memory, and leader care.
- Avoids streaks, rankings, guilt, and pressure.
- Distinguishes evidence from assumption.
- Produces one small next action.
