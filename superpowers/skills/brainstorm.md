# Brainstorm Skill

## Purpose

Generate, sort, and filter product ideas without drifting away from Donghaeng Room's MVP wedge.

## When To Use

- When exploring what to build next.
- When user feedback suggests a problem but not a solution.
- Before adding anything that might become chat, feed, content, ERP, payment, or analytics scope.

## Inputs Required

- User problem or beta observation.
- Target user: leader, member, or operator.
- Current context or evidence.
- Any known constraint: time, cost, privacy, platform, or beta urgency.

## Step-By-Step Process

1. Restate the user problem in one sentence.
2. List 5-8 possible responses, including at least one non-feature response.
3. Compare each idea against KakaoTalk, Naver BAND, YouVersion, Hallow/Glorify/Pray.com, and Planning Center/Church Center.
4. Mark ideas as build now, test manually, defer, or reject.
5. Keep only ideas that strengthen check-in, prayer memory, or leader care.
6. Select the smallest next experiment.

## Output Format

```md
## Brainstorm

Problem:
- ...

Ideas:
- ...

Differentiation Filter:
- ...

Recommended Next Experiment:
- ...
```

## Acceptance Criteria

- Includes at least one low-cost or manual option.
- Explicitly rejects ideas that imitate comparison products.
- Ends with one small next experiment.
- Does not authorize implementation by itself.
