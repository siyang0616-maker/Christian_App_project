# Workflow 07: Competitor Drift Audit

## Trigger

Roadmap review, feature creep concern, or any request that sounds like chat, feed, Bible content, devotional content, admin tooling, analytics, payment, push, upload, or native app work.

## Goal

Keep Donghaeng Room differentiated and focused on its wedge.

## Required Input

- Proposed roadmap item, feature, or current product area.
- User problem.
- Expected outcome.
- Current MVP stage.

## Agents To Call

- Entrepreneur
- Marketer
- Designer
- Developer

## Skills To Use

- competitor-differentiation
- roadmap-update

## Step-By-Step Process

1. State the proposed direction.
2. Compare it against KakaoTalk.
3. Compare it against Naver BAND.
4. Compare it against YouVersion.
5. Compare it against Hallow / Glorify / Pray.com.
6. Compare it against Planning Center / Church Center.
7. Decide whether the idea strengthens or weakens the wedge.
8. If useful, narrow the idea into a check-in, prayer memory, or leader care version.
9. Record deferred or rejected items.

## Decision Gate

Keep only ideas that make leaders care better, help prayer requests stay remembered, or make check-ins safer/easier. Defer anything that competes with the comparison products on their strongest ground.

## Files To Update

- `docs/COMPETITOR_DIFFERENTIATION.md` if the positioning rule changes.
- A filled copy of `superpowers/templates/decision-record.md` for important decisions.
- `docs/PROJECT_STATUS.md` if roadmap priorities change.
